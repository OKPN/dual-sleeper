# Dual Sleeper (デュアル・スリーパー)

**Dual Sleeper** は、Windows PC（特にAI学習サーバーやリモートPC）に最適な、知的で段階的な省電力（電源・モニター管理）システムです。

単なる「時間経過によるスリープ」ではなく、**「ユーザーの物理的な操作」**、**「ネットワークの通信速度」**、および**「GPU（ビデオカード）の動作プロセス」**の3つの軸をインテリジェントに監視し、状況に応じた最適な省電力を自動で実行します。

---

## 🌟 主な機能

### 1. 段階的な省電力（二段階スリープ）
*   **第一段階（モニターのみ消灯）:** 
    操作がなく、かつ通信量も少ない状態になると、まずはモニターの電源だけを切ります。PC本体は起動したままであるため、バックグラウンド処理やリモートサーバーとしての機能は維持されます。
*   **第二段階（システムをスリープ）:** 
    モニター消灯後、さらに低通信状態が一定時間続いた場合、PC本体をスタンバイ（スリープ）に移行させます。

### 2. 【高度な判定】AI学習とゲーム放置の自動判別（GPU保護機能）
*   **LoRA学習中（スリープ阻止）:**
    NVIDIA製GPUの負荷を監視します。現在GPUを使用しているプロセス名に `python.exe`（PyTorchなどの実行環境）が含まれる場合は、AI学習中とみなして**スリープを自動で阻止**します（モニターの消灯のみ行われます）。
*   **ゲーム放置中（スリープ実行）:**
    メニュー画面などでゲーム（`eldenring.exe` 等）がGPUを高負荷のまま無駄に回し続けている場合は、保護対象外とみなして**安全にスリープを実行**し、無駄な電気代をカットします。

### 3. 時間帯別のスリープモード切り替え
*   **夜間（デフォルト 0:00 〜 7:00）:** 
    長時間使わない時間帯のため、待機電力が完全にゼロになる **「休止状態（ハイバネート）」** に移行します。
*   **それ以外の時間帯:** 
    すぐに作業に復帰できるよう、復帰が高速な **「スタンバイ（スリープ）」** に移行します。

### 4. 長時間無操作時の「強制モニター消灯」
*   大容量のファイルダウンロードやリモート接続中で通信が発生していても、指定時間（デフォルト 15分）操作がない場合は「目の前に人はいない」と判断し、**画面（モニター）だけを先行して消灯**します。裏でのダウンロード処理等は中断されません。

### 5. Discord通知 ＆ スリープ直前キャンセル機能
*   PCが実際にスリープまたは休止状態に入る10秒前に、**DiscordのWebhookを通じてスマホ等へスリープ予告をプッシュ通知**します。
*   この10秒間のカウントダウン（猶予時間）中にマウスを動かすかキーを押すと、**スリープ移行が即座にキャンセル**され、通常稼働に戻ります（キャンセル時も通知されます）。

### 6. Tailscale（VPN）等の仮想通信の自動除外
*   VPN接続の常時キープアライブ（生存確認）パケットのせいで画面が消えなくなるのを防ぐため、`Tailscale` などの仮想ネットワークアダプターの通信量を監視対象から自動的に除外します。

## 📂 ファイル構成

*   `dual_sleeper.py`: メインのPythonプログラム。
*   `config.json.example`: 設定ファイルのひな形（Webhook等の秘匿情報を除外したもの）。
*   `config.json`: **[ローカル専用]** 実際の動作設定ファイル（`.gitignore` によりGit管理から自動除外されます）。
*   `requirements.txt`: 動作に必要な外部ライブラリ（`psutil`）を指定。
*   `run.bat`: ライブラリの準備からスクリプトの起動までをダブルクリックで行うバッチファイル。
*   `test_webhook.bat`: **[テスト用]** `config.json` に設定したDiscord Webhookが正しく動くか検証するバッチファイル。
*   `.gitignore`: `config.json` などの個人環境設定ファイルをGitにコミットしないための設定。

## 🚀 使い方

### 1. 準備（Pythonのインストール）
本スクリプトの実行には **Python** が必要です。あらかじめインストールし、環境変数 (PATH) に追加しておいてください。

### 2. 設定ファイルの作成（初回のみ）
1.  このフォルダ内にある `config.json.example` をコピーします。
2.  コピーしたファイルを **`config.json`** にリネームします。
3.  `config.json` をメモ帳などで開き、各種しきい値や、Discord通知用URLの設定を行います（下記参照）。
    ※`.gitignore` が設定されているため、`config.json` にプライベートなURLを記述しても、誤ってGitHub等のパブリックリポジトリにプッシュされる心配はありません。

### 3. 起動方法
1.  **`run.bat`** をダブルクリックして実行します。
2.  自動的に必要なライブラリ（`psutil`）がインストールされ、監視コンソールが立ち上がります。

### 4. 終了方法
*   起動したコマンドプロンプトの画面で `Ctrl + C` を押すか、ウィンドウを閉じてください。

## ⚙️ 設定オプション (`config.json`)

`config.json` をテキストエディタで開くことで、簡単に動作設定をチューニングできます。

| 設定項目 | 初期値 | 説明 |
| :--- | :--- | :--- |
| `idle_limit_seconds` | `10` | モニター消灯を検討し始める無操作時間（秒）。（※初期値はテスト用パラメータ） |
| `network_limit_kbs` | `20.0` | 通信が発生していないとみなす通信速度のしきい値（KB/s）。 |
| `network_check_duration_seconds` | `10` | 通信量がしきい値以下であるかを確認し続ける判定時間（秒）。 |
| `check_interval_seconds` | `5` | PCの稼働状態をチェックする周期（秒）。 |
| `standby_after_monitor_off_seconds` | `10` | モニターが消灯したあと、PC本体をスリープさせるまでの待機時間（秒）。 |
| `hibernate_start_hour` | `4` | 夜間休止状態（ハイバネート）を適用する開始時刻（時）。 |
| `hibernate_end_hour` | `11` | 夜間休止状態（ハイバネート）を適用する終了時刻（時）。 |
| `force_monitor_off_idle_seconds` | `900` | 通信の有無に関わらず、無操作がこの秒数続いた場合にモニターを強制オフにする時間（秒）。（例: 900秒 ➔ 15分） |
| `discord_webhook_url` | `""` | DiscordのWebhook URL。空のままだとDiscord通知は無効になります。 |
| `sleep_pending_seconds` | `10` | Discordへスリープ予告を送信してから、実際にスリープするまでの猶予時間（秒）。 |
| `gpu_protect_processes` | `["python.exe", "python"]` | スリープを阻止したい（保護したい）現在GPUを使用しているプロセス名のリスト。 |
| `gpu_limit_percent` | `10` | GPU使用率がこの％以上で、かつ上記保護対象プロセスがGPUを使用している場合にスリープを阻止します。 |

## 🔔 Discord Webhookの設定とテスト方法

1.  通知を受け取りたいDiscordチャンネルの **「設定 (歯車マーク)」** ➔ **「連携サービス」** ➔ **「ウェブフック」** を選択します。
2.  **「新しいウェブフック」** を作成し、**「ウェブフック URL をコピー」** をクリックします。
3.  `config.json` を開き、`"discord_webhook_url"` の中にダブルクォーテーションで囲んで貼り付けます。
    ```json
    "discord_webhook_url": "https://discord.com/api/webhooks/..."
    ```
4.  設定完了後、フォルダー内にある **`test_webhook.bat`** をダブルクリックして実行します。
5.  設定したDiscordチャンネルにテスト通知が届くか確認してください。届いていれば連携は無事に完了しています！
